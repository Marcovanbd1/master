The Engine of Trust.

We believe that privacy is the prerequisite for honesty. Our architecture is designed to protect the user and the institution by adhering to a strict Zero-Data standard.

1. Zero-Data Architecture

Most health apps trade service for data. We do not. Our system is built on a "No-Account, No-Tracking" architecture that renders the user completely invisible to us.

  • No PII Collection: We do not request names, emails, IP addresses, or NHS numbers.
  • Ephemeral Sessions: The user's interaction exists only for the duration of the session. Once the browser is closed, the personal context vanishes.
  • Client-Side Privacy: Emotional input is processed to generate the playlist, but the raw text is never stored in a persistent user profile.
PROTOCOL: [Input] -> [ANCI Translation] -> [Playlist Output] -> [Session Wipe]

2. ANCI (Autonomous Non-Clinical Intelligence)

Our core engine is not a chatbot and it does not "hallucinate." It is a specialized translation layer designed to map unstructured distress to structured support paths.

  • Translation, Not Diagnosis: ANCI identifies themes (e.g., "financial worry," "sleep loss") but never assigns a clinical diagnostic label.
  • The Safety Deflection Loop: If ANCI detects specific high-risk keywords (e.g., immediate self-harm), it triggers a hard-coded safety override, directing the user to emergency services immediately.

3. DSI (Demand Shaping Intelligence)

While we protect the individual, we empower the system. DSI is the aggregate, anonymous output of the platform.

By stripping away identity, we are left with pure signal. We aggregate broad themes (e.g., "30% increase in housing anxiety in Region X") to provide institutions with a "Civic Radar" for resource allocation.

Built for the "Quiet Middle." Validated for the Enterprise.
